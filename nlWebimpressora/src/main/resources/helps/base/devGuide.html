Guia para desenvolvimento

Para desenvolver um HELP, é necessário criar uma pasta no seu computador localizado em /workspace_eclipse/helps e por a estrutura de arquivos presente /src/main/resources/helps lá dentro. Esse passo é necessário pois não temos como refletir as mudanças nos arquivos de forma instantânea sem saber o lugar comum de onde buscar os arquivos. Também não é possível apontar para os arquivos presentes no workspace pois o runtime do java está localizado no Wildfly que é completamente desconexo do workspace atual.

Abaixo há o comando que já irá fazer a cópia dos helps atuais para a pasta necessária

ln -s <caminho_seu_workspace>/src/main/resources/helps /workspace_eclipse/helps

Lembrando então que é necessário por os arquivos desenvolvidos dentro de /src/main/resources/helps quando finalizados para que sejam mandados para o Git e sejam postos dentro da build, caso contrário, esses não irão estar presentes do arquivos WAR

Para fazer o link do Help com a página, deve ser chamado a hook useNlHelp passando como primeiro parâmetro o caminho a partir da pasta /src/main/resources/helps mencionada anteriormente.

Sobre os helps

Os helps são desenvolvidos em HTML Puro com o auxilio do JQuery e do Bootstrap 5, de forma que assim tempos muitas funcionalidades já prontas, estilizadas, e com o melhor de tudo, documentação a vontade!

Podemos utilizar então todas as funcionalidades do bootstrap como abas, menus, collapses, tooltips, modais, popovers, dropdowns, cards, botões, etc.

Para exemplos de componentes, acessar o help da página /nl/nlTestaComponente
